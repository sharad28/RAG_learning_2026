RAG Pipeline — Learning Plan

5-phase curriculum from fundamentals to production-grade RAG with live retrieval scoring demo.

Weeks 1–2 · Beginner
Foundations
Build the mental model — understand embeddings, vector spaces, and why RAG exists before writing a single line of code.
What is RAG?
Retrieval-Augmented Generation: augment LLM answers with relevant documents retrieved at query time.
Embeddings & vector spaces
How text is turned into high-dimensional vectors; semantic similarity as cosine distance.
Vector databases
FAISS, Chroma, Pinecone, Weaviate — how they store and search vectors efficiently (HNSW, IVF).
Chunking strategies
Fixed-size vs. sentence vs. semantic chunking; chunk overlap; why chunk size affects retrieval quality.
LLM prompting basics
System prompts, context windows, how retrieved docs are injected into prompts.
Cosine & dot-product similarity
The math behind retrieval scoring; L2 norm, cosine similarity formula, when to use each.

Weeks 3–5 · Intermediate
Build a basic RAG pipeline
Implement a working end-to-end pipeline: ingest documents, embed them, store in a vector DB, retrieve, and generate.
Document ingestion
Load PDFs, HTML, Markdown. Use LangChain loaders or LlamaIndex readers. Handle encoding issues.
Embedding models
OpenAI text-embedding-3, Sentence-BERT, BGE, Nomic. Trade-offs: speed, cost, accuracy.
FAISS / Chroma setup
Create an index, add vectors, query top-k nearest neighbours. Persist to disk.
Retrieval + generation
Stuff documents into prompt context. Handle context window limits gracefully.
Metadata filtering
Filter by date, source, category before or after ANN search to narrow results.
BM25 + hybrid search
Combine keyword BM25 with dense vector retrieval using RRF (Reciprocal Rank Fusion).

Weeks 6–9 · Advanced
Advanced RAG techniques
Go beyond naive RAG — improve retrieval quality and answer accuracy with query transformation, re-ranking, and smarter indexing.
Query rewriting & HyDE
Hypothetical Document Embeddings: generate a fake answer to the query, embed it, and retrieve using that embedding.
Multi-query retrieval
Generate 3–5 variants of the query, retrieve for each, deduplicate and merge results for better recall.
Contextual compression
Use an LLM to compress retrieved chunks to only the parts relevant to the question before generation.
Re-ranking (cross-encoders)
Cohere Rerank, BGE-Reranker: score (query, doc) pairs directly for much higher precision than bi-encoder ANN.
Parent-document retrieval
Index small chunks but return their parent (larger) chunk for generation — best of both recall and context.
RAPTOR (recursive clustering)
Summarise clusters of documents, index summaries too — enables multi-hop reasoning over long corpora.
GraphRAG
Build a knowledge graph from documents, retrieve via graph traversal — strong on relational/multi-hop queries.
Agentic RAG
LLM decides when and what to retrieve, can retry or reformulate — useful for complex, multi-step reasoning.

Weeks 13–16 · Expert
Production & optimisation
Serve RAG at scale — caching, observability, cost control, continuous eval, and fine-tuning retrievers.
Semantic caching
Cache query embeddings and responses. If a near-duplicate query arrives, return cached answer — saves cost & latency.
Streaming responses
Stream tokens as the LLM generates while retrieval completes in parallel. Reduce perceived latency.
Observability & tracing
LangSmith, Arize Phoenix, Langfuse — trace every retrieval call, log scores, debug failures.
Fine-tuning embeddings
Domain-adapt embedding models using sentence-transformers with synthetic Q&A pairs from your corpus.
Incremental indexing
Update the vector index as documents change without rebuilding from scratch — document versioning patterns.
Guard rails & safety
Detect hallucinations, refuse out-of-scope queries, add citation markers back to source documents.

16 weeks total
Week-by-week timeline
A structured schedule you can follow at ~10–15 hrs/week.
Wk 1–2
Foundations — embeddings, vectors, RAG intro
Read the RAG paper, watch 3B1B, build a toy cosine similarity calculator in Python.
Wk 3–4
First pipeline — LangChain + Chroma + OpenAI
Ingest a PDF corpus, embed with text-embedding-3-small, query top-5, generate answer. Deploy locally.
Wk 5
Hybrid search — BM25 + dense + RRF
Add BM25 retrieval, implement RRF fusion, compare results vs. dense-only.
Wk 6–7
Advanced retrieval — HyDE, MultiQuery, Parent-Doc
Implement each technique on your corpus; measure recall improvement manually.
Wk 8–9
Re-ranking + GraphRAG intro
Cohere Rerank API, BGE-Reranker locally. Read GraphRAG paper, run the Microsoft repo.
Wk 10–11
Evaluation — RAGAS + TruLens
Build an eval dataset (100 Q&A pairs). Score context precision, faithfulness, answer relevance. Establish a baseline.
Wk 12
Iterative improvement loop
Run evals, change one component (chunk size, retriever, reranker), re-run evals, track deltas.
Wk 13–14
Production hardening — caching, tracing, streaming
Add semantic cache (GPTCache or Redis), instrument with LangSmith, stream tokens to a FastAPI endpoint.
Wk 15–16
Fine-tune embeddings + capstone project
Generate synthetic pairs, fine-tune BGE on your domain. Ship a complete RAG app with eval dashboard.
